- Chronology: EvidenceRecord ordering is now derived solely from each artifact's own timestamp, validated as ISO 8601 / RFC 3339 with an explicit timezone, normalized to a canonical UTC value ending in Z. Malformed, naive (no timezone), or non-string timestamps are rejected. Records are sorted by (parsed UTC timestamp, evidence_id); manifest-provided ordering hints are never used.
- Ground-truth isolation hardening: src/continuity_ai/artifact_io.py (validate_production_artifact_root, open_production_artifact) and src/continuity_ai/ingestion.py now reject the forbidden test_only directory and ground_truth.json filename case-insensitively, and validate_production_artifact_root scans the full artifact subtree so a forbidden path is rejected even if the manifest never references it. The resolved-path containment boundary is preserved.
- URI hardening: ingestion rejects POSIX absolute paths, Windows absolute and drive-relative paths (e.g. C:/outside.txt, C:outside.txt) using pathlib semantics for both POSIX and Windows path forms regardless of host platform, backslashes, parent traversal, paths escaping through resolution, and forbidden ground-truth names case-insensitively. Duplicate URIs are rejected case-insensitively.
- Manifest schema hardening: unexpected top-level and per-entry fields, non-hexadecimal or wrong-length sha256 values, invalid UTF-8, read errors, and malformed JSON are all rejected, wrapped consistently as ArtifactIngestionError. The project field may be any non-empty string (ingestion is not hardcoded to Project Aurora).
- Empty-content hardening: ingestion fails closed when parsed, normalized content carries no meaningful evidence: empty Markdown, PDF with no extractable text, spreadsheets with no meaningful cell values, emails with empty subject and body (the literal "Subject:" label is never treated as evidence), and calendars with no meaningful SUMMARY/LOCATION/DESCRIPTION (missing ICS fields are omitted, never serialized as the string "None"). No OCR or document recovery is implemented.
- Fixture determinism fix: aurora_fixture.py's XLSX writer previously left docProps/core.xml's modified timestamp non-deterministic, because openpyxl's save_workbook() unconditionally overwrites workbook.properties.modified with the real wall-clock time during save regardless of what is set beforehand; this only surfaced as an intermittent test failure when two saves straddled a one-second boundary. The writer now re-pins the fixed modified timestamp directly in the saved XML after save, before zip normalization, making budget_v4.xlsx (and therefore evidence_manifest.json, which embeds its checksum) deterministic across independent generations.
